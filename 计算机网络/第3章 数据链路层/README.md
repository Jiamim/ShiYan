数据链路层
==========
##使用的信道
数据链路层使用的信道有以下两种类型：
* 点对点信道（PPP协议）
* 广播信道（共享信道协议）

##三个基本问题
1. 封装成帧
2. 透明传输
3. 差错控制

##使用广播信道的数据链路层
###局域网的主要优点：
1. 具有广播功能
2. 便于系统的扩展和逐渐地演变
3. 提高了系统的可靠性、可用性和生存性


##帧
数据链路层的通信单位是帧
###进一步分层
IEEE将数据链路层进一步划分：
* 逻辑链路控制（Logical Link Control），即LLC。
* 媒体接入控制（Media Access Control）,即MAC。比如：
  * 以太网MAC
  * 令牌环MAC
  * 令牌总线MAC
  
MAC层与物理层相邻，不同的物理层介质对应着不同类型的MAC。  
LLC层在MAC层之上与网络层相邻，不同的MAC层都对应着统一的LLC。
>目前，LLC的作用已经消失了。很多厂商的适配器上就只有MAC协议
###MAC地址
又称数据链路地址、物理地址。不同的物理层，它们的MAC地址的类型是不同的。

常见的是48位的只是以太网MAC。

##IEEE802标准
|IEEE标准|描述
|---|----
|802.2|逻辑链路控制**LLC**
|802.3|CSMA/CD（以太网）
|802.4|令牌总线
|802.5|令牌环
|802.6|分布队列双总线DQDB
|802.8|光纤分布数据接口FDDI
|802.11|无线LAN（WLAN）
