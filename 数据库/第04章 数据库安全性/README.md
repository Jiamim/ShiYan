数据库安全性
============
##计算机安全性
计算机系统的三类安全性问题
- 技术安全
- 管理安全
- 政策法律

##数据库安全性控制
###用户标识与鉴别
这是系统提供的最外层安全保护措施。  

常用方法有：
- 用户标识（User Identification）
- 口令（Password）

###存取控制
存取控制机制主要包括两部分：
- 定义用户权限，并将用户权限登记到数据字典中
- 合法权限检查

###自主存取控制（DAC）方法
主要通过SQL语句的GRANT语句和REVOKE语句实现。

用户权限主要有两个要素构成：
- 数据库对象
- 操作类型

在数据库中，定义存取权限称为**授权**。
###强制存取控制（MAC）方法
>自主存取控制能够通过**授权机制**有效地控制对敏感数据的存取。  
MAC适用于那些对数据有严格而固定密级分类的部分，如军事部门或政府部门

在MAC中，DBMS所管理的全部实体被分成主体和客体两大类。

|实体|定义|包括内容
|----|----|---
|主体|是系统中的活动实体|即包括DBMS所管理的实际用户，也包括代表用户的各进程
|客体|是系统中的被动实体，是受主体操纵的|包括文件、基本表、索引、视图等

对于主体和客体，DBMS为它们每个实例指派一个**敏感度标记**。
* 主体的敏感度标记被称为**许可证基本**
* 客体的敏感度标记称为**密级**。
  
敏感度标记被分成若干级别，如：**绝密**、**机密**、**可信**、**公开**等

###GRANT
该语句可以由DBA或数据所有者发出。  
GRANT语句的一般格式为：
```sql
	GRANT <权限> [,<权限> ]...
	ON <对象类型> <对象名> [,<对象类型> <对象名>]...
	TO <用户> [,<用户> ]...
	[WITH GRANT OPTION];
```
权限字段包括SELECT、INSERT、UPDATE等。ALL PRIVILEGES表示所有权限。   
对象类型比如TABLE，则对象名为表名。  
用户字段，如果是PUBLIC表示全部用户。  
>选项WITH GRANT OPTION如果存在，则表明获得了该权限的用户可以授予该权限给其他用户。

举例：
```sql
	GRANT UPDATE(Sno),SELECT
	ON TABLE Student，Sc
	TO U4
	WITH GRANT OPTION;
```
###REVOKE
回收权限。由DBA或其他授权者发出。  
REVOKE语句的一般格式为：
```sql
GRANT <权限> [,<权限> ]...
	ON <对象类型> <对象名> [,<对象类型> <对象名>]...
	FROM <用户> [,<用户> ]...[CASCADE|RESTRICT];
```
CASCADE表示级联。比如用户U1把UPDATE权限授予了U2，如果指定了选项CASCADE，则收回U1的UPDATE权限之后，U2的UPDATE权限也收回。
